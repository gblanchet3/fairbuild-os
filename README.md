# FairBuild OS

A file-based operating system for running FairBuild AI — company knowledge, customer intel, sales playbooks, campaigns, and content, all living in one repo that Claude Code can read, write, and reason about.

**This repo itself is the demo.** The [proposal](00-proposal/gvb-sprint-proposal.md) you received is a markdown file in `00-proposal/`. The folders below are scaffolded for the sprint — we fill them in together.

---

## What's here

| Folder | Purpose |
|---|---|
| [`00-proposal/`](00-proposal/) | GVB's 2-week sprint proposal |
| [`01-company/`](01-company/) | FairBuild wiki — what we do, why, team, fundraising |
| [`02-icp-and-persona/`](02-icp-and-persona/) | Target customer, buyer persona, pain language |
| [`03-voice-and-tone/`](03-voice-and-tone/) | Brand voice, messaging pillars, words we use / don't use |
| [`04-product/`](04-product/) | Product overview, demo script, "oh wow" moments |

| [`06-sales-playbook/`](06-sales-playbook/) | Demo script, objection handling, contract risk insights |
| [`07-content-library/`](07-content-library/) | LinkedIn posts, ad copy archive, organized by campaign |
| [`08-campaigns/`](08-campaigns/) | Live campaign dashboards — what's running, what's working |
| [`09-goals/`](09-goals/) | Annual + sprint goals |

---

## How to use this

**Passive (this week):** just browse the folders. This is FairBuild OS on day 1.

**Active (starting day 1 of the sprint):** clone this repo to your laptop, open it in Claude Code, start filling in the stubs. Every folder has a README with the first prompt to run.

---

## The point

Most founders ask Claude the same question 50 different ways because they have nowhere to put the answer. This repo is the answer.

Ask Claude once: *"Write me three LinkedIn ad variants for framers."*
Save the output to `07-content-library/ad-copy/`.
Next month, ask: *"Using our ad copy archive and what we learned about framers, draft new variants for LBM dealers."*

That's the compounding. That's the OS.
